This blog is generated by a program written in **Rocq** (formerly Coq) and extracted to C++ through [Bloomberg's Crane](https://github.com/bloomberg/crane). The Markdown you are reading was parsed and rendered by that extracted binary. Nothing in the pipeline is interpreted at runtime; everything is compiled ahead of time and linked into a single deterministic executable.

## What is actually checked

I want to be precise about the word *verified*, because it is easy to oversell. What Rocq and Crane give us here is **type-level** discipline: every function is total, recursion is fuel-bounded, and extraction produces well-typed C++ that operates on primitive strings without panics. The build is reproducible: the same input tree produces byte-identical output.

What is not proven is the semantics of the parser or renderer. There is no theorem stating that my Markdown reader is sound against CommonMark, or that the HTML escape function covers every dangerous byte. Those are real obligations and they remain future work. For now, the renderer is small enough to read in one sitting, and that is a different kind of confidence.

## A taste of the source

Here is the character escape routine, more or less verbatim from `src/Logic.v`:

```coq
Definition html_escape_char (s : string) (pos : int) : string :=
  let ch := PrimString.get s pos in
  if int_eqb ch ch_amp then "&amp;"
  else if int_eqb ch ch_lt then "&lt;"
  else PrimString.sub s pos 1%int63.
```

The inline parser handles the usual suspects: `[label](url)` links, backtick code spans, and now *emphasis* and **strong** with proper nesting. Block-level support covers fenced code with an optional language tag, H2 and H3 headings, and image lines on their own. That is the whole grammar. Anything more exotic is either dropped or rendered as plain text.

## Why bother

Writing a generator this way is slower than reaching for an off-the-shelf tool, and the output is deliberately modest. The point is the shape of the artifact: a small core, a narrow interface, and a build I can reason about. Future posts will expand the grammar and, eventually, prove the parts worth proving.
